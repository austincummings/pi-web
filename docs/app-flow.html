1 Setup & wiring

  • Custom-element imports (import "./pi-tool.ts") are bare side-effect imports on purpose — import type would get tree-shaken and silently break customElements.define().
  • Grabs DOM handles ($transcript, composer, picker, dialog…).
  • Exposes window.piweb.registerToolRenderer for extensions; registers the built-in edit → colored-diff renderer.

2 Composer input flow

The <pi-composer> element owns the textarea, submit, history, images, and working spinner. app.ts reacts to its events:

  • pi-input → updateAc() + applyThinkingBorder() + pushEditorText()
  • pi-submit → runInput(text, images)
  • pi-dequeue → restoreQueue()

runInput() is the central dispatcher:

!  / !!   → POST /bash   (shell; !! excludes from context)
/         → runCommand()  else POST /command
otherwise → POST /prompt

3 Autocomplete updateAc / acceptAc

One typeahead popup with several modes:

command

/ slash palette (built-in + extension commands), fuzzy-filtered.

dir

/new <dir> directory completion.

file

@file mentions spliced mid-line.

bash

!-command file completion, force-opened by Tab.

ext

Extension providers via /autocomplete.

Async sources use an acReq sequence guard so a slow fetch can't clobber newer keystrokes.

4 Slash commands runCommand

Client-handled commands map to pickers/overlays: /resume, /new, /model, /tree, /fork, /clone, /session, /trust, /export, /share… Most fetch data then open a picker (openListPicker / openPicker / modelPicker).

5 Thread model

  • Active thread lives in the URL (?thread=<id>) — survives refresh, shareable. activeThreadId mirrors it.
  • gotoThread() pushes URL + reopenStream(). popstate handles back/forward.
  • First load: open the URL's thread, or POST /threads to create a fresh one.

7 Rendering extension UI renderNode

A serializable component-tree renderer: Button/Input/Frame are interactive (post /action); everything else (Box/Text/Markdown/Image…) goes to the shared renderStaticNode. Frame nodes become sandboxed <pi-frame> iframes whose actions/notifies bubble up to document listeners → host or toasts.

8 Global keyboard

  • Esc precedence: AC → picker → overlay → interrupt working agent → cancel bash → restore queue.
  • Alt+T toggle thinking · Alt+O expand all tools · Shift+Tab cycle reasoning level · Alt+↑ restore queue.

The recurring pattern: custom elements own their local state/DOM; app.ts is the router between their events, HTTP POSTs, and the SSE frame stream.
